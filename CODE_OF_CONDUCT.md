# Code of Conduct

## Our standard

CVE Lite CLI aims to be a respectful, practical, and constructive open source project.

Examples of behavior that help create a positive environment include:

- being respectful in discussions
- assuming good intent
- giving constructive feedback
- focusing on technical substance
- being welcoming to new contributors
- keeping disagreements professional

Examples of unacceptable behavior include:

- harassment or personal attacks
- insulting or demeaning comments
- deliberate intimidation
- trolling or bad-faith engagement
- publishing private information without permission
- any conduct that makes participation unsafe or hostile

## Project expectations

This project values:

- honest technical discussion
- respectful disagreement
- clear and actionable feedback
- community-helpful contributions

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, or other contributions that violate this Code of Conduct.

Repeated or serious violations may result in temporary or permanent exclusion from the project space.

## Scope

This Code of Conduct applies in project spaces, including issues, pull requests, discussions, and other public interactions related to the project.

## Reporting

If you experience or witness unacceptable behavior, report it through the project contact path.
